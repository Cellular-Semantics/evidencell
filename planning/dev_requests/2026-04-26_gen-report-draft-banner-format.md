# Dev request: Fix draft warning banner format in gen-report.md

**Date:** 2026-04-26  
**Blocked step:** `workflows/gen-report.md` — Section 1 (Report header), draft status banner  
**Severity:** Medium — causes validation hook failures on all agent-generated reports for draft graphs

---

## What is blocked

The synthesis subagent prompt in `gen-report.md` (line 96) instructs the model to emit the draft warning as a Markdown blockquote:

```
> ⚠ Draft mappings. Evidence is atlas-metadata only unless otherwise noted.
> All edges require expert review before use.
```

The pre-write validation hook (`validate_mapping_hook.py`) rejects any blockquote (`>` line) in a report file that is **not** followed by a `<!-- quote_key: X -->` attribution annotation. The draft banner is not a literature quote and has no quote key, so the hook rejects every report generated from a draft graph.

This caused all 9 sexually_dimorphic reports generated by background agents to fail on the first Write attempt.

---

## What needs to change

**File:** `workflows/gen-report.md`, line 96–97

**Current:**
```markdown
> ⚠ Draft mappings. Evidence is atlas-metadata only unless otherwise noted.
> All edges require expert review before use.
```

**Proposed fix:**
```markdown
**⚠ Draft mappings. Evidence is atlas-metadata only unless otherwise noted. All edges require expert review before use.**
```

Bold inline text (`**...**`) is not a blockquote and does not trigger the hook. The visual emphasis is preserved.

The existing reports (`reports/sexually_dimorphic/*.md`) have already been manually updated to use `**⚠ ...**` format.

---

## What was tried

- Reports were manually rewritten in the current session to use `**⚠ ...**` — this passes the hook.
- Direct edit to `workflows/gen-report.md` is blocked by the curation-mode pre-edit hook (workflows/ is out of scope).

---

## Proposed surface

Single-line change in `workflows/gen-report.md` at the point where the subagent prompt instructs the banner format. No schema changes needed.
